# Design Decisions & Trade-offs

## Prisma ORM

* Chosen for type safety and developer productivity
* Trade-off: Less control vs raw SQL

## PostgreSQL

* Reliable relational database
* Supports aggregation queries for insights

## Monolithic Backend

* Faster to build and simpler deployment
* Trade-off: Less flexible than microservices

## Docker

* Ensures consistency across environments
* Trade-off: Slight setup complexity
